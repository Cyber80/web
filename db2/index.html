แนวข้อสอบระบบฐานข้อมูล 50 ข้อ

คำชี้แจง: เลือกคำตอบที่ถูกต้องที่สุดเพียงข้อเดียว แล้วกดปุ่ม แสดงเฉลย เพื่อดูคำตอบ

หมวดที่ 1: พื้นฐานฐานข้อมูล

1. การออกแบบฐานข้อมูลมีวัตถุประสงค์หลักข้อใด?
คำตอบที่ถูกต้อง: ลดความซ้ำซ้อนของข้อมูล
2. ข้อใดเป็นผลลัพธ์ของการออกแบบฐานข้อมูลที่ดี?
คำตอบที่ถูกต้อง: ประสิทธิภาพสูง
3. ปัญหาหลักของการออกแบบฐานข้อมูลที่ไม่ดีคือ?
คำตอบที่ถูกต้อง: ข้อมูลซ้ำซ้อน
4. การรักษาความถูกต้องของข้อมูลเรียกว่าอะไร?
คำตอบที่ถูกต้อง: Data Integrity
5. การออกแบบฐานข้อมูลที่ดีช่วยเรื่องใด?
คำตอบที่ถูกต้อง: ประหยัดทรัพยากร

หมวดที่ 2: ขั้นตอนการออกแบบฐานข้อมูล

6. ขั้นตอนแรกของการออกแบบฐานข้อมูลคืออะไร?
คำตอบที่ถูกต้อง: Requirements Analysis
7. เครื่องมือใดใช้ในการออกแบบเชิงแนวคิด (Conceptual Design)?
คำตอบที่ถูกต้อง: ER Diagram
8. ในการออกแบบเชิงตรรกะ ต้องทำสิ่งใด?
คำตอบที่ถูกต้อง: แปลง Entity เป็นตาราง
9. การออกแบบเชิงกายภาพ (Physical Design) ควรพิจารณาเรื่องใด?
คำตอบที่ถูกต้อง: Data Type และ Index
10. ขั้นตอนใดที่จะนำแบบออกแบบไปใช้งานจริง?
คำตอบที่ถูกต้อง: Implementation
11. การทดสอบและบำรุงรักษามีเป้าหมายเพื่ออะไร?
คำตอบที่ถูกต้อง: ตรวจสอบและปรับปรุงระบบ
12. ตัวอย่างกิจกรรมใน Requirements Analysis คืออะไร?
คำตอบที่ถูกต้อง: สัมภาษณ์ผู้ใช้งาน
13. Cardinality ใน ER Diagram หมายถึงอะไร?
>
คำตอบที่ถูกต้อง: ความสัมพันธ์ระหว่าง Entity
14. ในการออกแบบ Logical Design ต้องกำหนดอะไรให้ชัดเจน?
คำตอบที่ถูกต้อง: Data Type และ Foreign Key
15. การสร้าง INDEX ช่วยเรื่องอะไร?
คำตอบที่ถูกต้อง: เพิ่มความเร็วการค้นหา

หมวดที่ 3: คีย์และความสัมพันธ์

16. Primary Key มีหน้าที่อะไร?
คำตอบที่ถูกต้อง: ระบุแถวให้ไม่ซ้ำกัน
17. Foreign Key ใช้ทำอะไร?
คำตอบที่ถูกต้อง: เชื่อมโยงกับ Primary Key ของตารางอื่น
18. Super Key แตกต่างจาก Candidate Key อย่างไร?
คำตอบที่ถูกต้อง: Super Key อาจประกอบด้วยแอตทริบิวต์เพิ่มเติม
19. Composite Key คืออะไร?
คำตอบที่ถูกต้อง: คีย์ที่ประกอบด้วยหลายฟิลด์
20. ความสัมพันธ์แบบ One-to-Many (1:M) หมายความว่าอย่างไร?
คำตอบที่ถูกต้อง: หนึ่งฝั่งมีความสัมพันธ์กับหลายฝั่ง
21. เมื่อมีความสัมพันธ์แบบ Many-to-Many ควรแก้ไขอย่างไร?
คำตอบที่ถูกต้อง: สร้างตารางเชื่อมกลาง (Junction Table)
22. ข้อใดเป็น Candidate Key ในตาราง Students(StudentID, CitizenID, Email)?
คำตอบที่ถูกต้อง: ทั้งหมดที่กล่าวมา
23. ข้อใดไม่ใช่หน้าที่ของ Primary Key?
คำตอบที่ถูกต้อง: เก็บค่าซ้ำได้
24. Weak Entity แตกต่างจาก Strong Entity อย่างไร?
คำตอบที่ถูกต้อง: Weak Entity ไม่มี Primary Key ของตัวเอง
25. Composite Attribute คืออะไร?
คำตอบที่ถูกต้อง: แอตทริบิวต์ที่สามารถแบ่งย่อยได้ (เช่น ชื่อเต็ม)

หมวดที่ 4: คำสั่ง SQL (SQL Commands)

26. คำสั่งใดใช้เพื่อดึงข้อมูลจากฐานข้อมูล?
คำตอบที่ถูกต้อง: SELECT
27. คำสั่งใดใช้เพิ่มแถวใหม่ในตาราง?
คำตอบที่ถูกต้อง: INSERT INTO
28. คำสั่งใดใช้แก้ไขข้อมูลที่มีอยู่แล้ว?
คำตอบที่ถูกต้อง: UPDATE
29. คำสั่งใดใช้ลบแถวข้อมูลในตาราง?
คำตอบที่ถูกต้อง: DELETE FROM
30. หากต้องการลบตารางทั้งหมดใช้คำสั่งใด?
คำตอบที่ถูกต้อง: DROP TABLE
31. การเลือกทุกคอลัมน์จากตารางใช้สัญลักษณ์ใด?
คำตอบที่ถูกต้อง: *
32. ฟังก์ชันใดใช้สำหรับนับจำนวนแถว?
คำตอบที่ถูกต้อง: COUNT()
33. การกรองข้อมูลใน SELECT ใช้คำสั่งใด?

คำตอบที่ถูกต้อง: WHERE
34. คำสั่งใดใช้จัดกลุ่มผลลัพธ์เพื่อใช้ Aggregate Function?
คำตอบที่ถูกต้อง: GROUP BY
35. คำสั่งใดเรียงลำดับผลลัพธ์จากน้อยไปมาก?
คำตอบที่ถูกต้อง: ORDER BY ... ASC
36. คำสั่งใดใช้สร้างตารางใหม่?
คำตอบที่ถูกต้อง: CREATE TABLE
37. คำสั่งใดคือ DDL (Data Definition Language)?
คำตอบที่ถูกต้อง: CREATE, ALTER, DROP
38. คำสั่งใดใช้สำหรับค้นหาข้อมูลที่มีรูปแบบบางส่วน เช่น ขึ้นต้นด้วย 'S'?
คำตอบที่ถูกต้อง: WHERE ... LIKE 'S%'
39. คำสั่ง JOIN ใดที่คืนเฉพาะแถวที่ตรงกันทั้งสองตาราง?
คำตอบที่ถูกต้อง: INNER JOIN
40. คำสั่งใดใช้รีเซ็ตข้อมูลทั้งหมดของตาราง (ลบข้อมูลแต่เก็บโครงสร้าง)?
คำตอบที่ถูกต้อง: TRUNCATE TABLE

หมวดที่ 5: Access, Data Types และ Data Dictionary

41. Microsoft Access เป็นประเภทใด?
คำตอบที่ถูกต้อง: DBMS
42. Object ใดไม่มีใน Access?
คำตอบที่ถูกต้อง: Worksheet
43. Query ใน Access ใช้งานได้อย่างไร?
คำตอบที่ถูกต้อง: กรองข้อมูล, คำนวณ, ค้นหา, รวมจากหลายตาราง
44. Form ใน Access ใช้เพื่ออะไร?
คำตอบที่ถูกต้อง: ป้อนและแสดงข้อมูล
45. Report ใน Access ใช้สำหรับ?
คำตอบที่ถูกต้อง: แสดงข้อมูลสรุปและพิมพ์รายงาน
46. ชนิดข้อมูลใดเหมาะสำหรับข้อความยาวใน Access?
คำตอบที่ถูกต้อง: Memo (Long Text)
47. ชนิดข้อมูลใดใช้เก็บรูปภาพใน Access แบบดั้งเดิม?
คำตอบที่ถูกต้อง: OLE Object
48. Data Dictionary มีประโยชน์หลักเรื่องใด?
คำตอบที่ถูกต้อง: คำอธิบายฟิลด์และชนิดข้อมูล
49. Normalization มีจุดประสงค์เพื่ออะไร?
คำตอบที่ถูกต้อง: ลดความซ้ำซ้อนและรักษาความถูกต้องของข้อมูล
50. ข้อใดเป็นหลักการสำคัญของการออกแบบฐานข้อมูลที่ดี?
คำตอบที่ถูกต้อง: Integrity, Efficiency, Scalability
