บทเรียน Microsoft Access 2021
ระบบจัดการข้อมูลชมรมโรงเรียน

สำหรับนักเรียนระดับชั้นมัธยมศึกษาปีที่ 5

โดย: ครูเจษฎา ชมเชย

ภาพรวมรายวิชา

ในยุคดิจิทัลนี้ ข้อมูลมีบทบาทสำคัญมากในทุกๆ ด้านของชีวิต ไม่ว่าจะเป็นการเรียน, การทำงาน, หรือแม้แต่การเล่นเกม! การที่เราสามารถจัดการข้อมูลจำนวนมากๆ ได้อย่างเป็นระบบ จะช่วยให้เราทำงานได้เร็วขึ้น ลดความผิดพลาด และสามารถนำข้อมูลเหล่านั้นมาใช้ประโยชน์ได้จริง

รายวิชา Microsoft Access 2021 นี้ จะพานักเรียนไปรู้จักกับโปรแกรม Access ซึ่งเป็นเครื่องมือทรงพลังในการสร้างและจัดการฐานข้อมูล เราจะเรียนรู้ตั้งแต่พื้นฐานว่าฐานข้อมูลคืออะไร ไปจนถึงการสร้างระบบเล็กๆ อย่าง "ระบบจัดการข้อมูลชมรมโรงเรียน" ที่นักเรียนจะได้ออกแบบและลงมือทำจริง!

เป้าหมายการเรียนรู้:

  • เข้าใจหลักการพื้นฐานของฐานข้อมูลและ Microsoft Access
  • สร้างและจัดการฐานข้อมูล, ตาราง (Tables), แบบสอบถาม (Queries), ฟอร์ม (Forms) และรายงาน (Reports) ได้อย่างมีประสิทธิภาพ
  • นำความรู้ไปประยุกต์ใช้ในการจัดเก็บและเรียกใช้ข้อมูลสำหรับงานง่ายๆ ในชีวิตประจำวันหรือการเรียน
  • ตระหนักถึงความสำคัญของการจัดการข้อมูลอย่างเป็นระบบ

โลโก้ Microsoft Access 2021

บทที่ 1: บทนำสู่ฐานข้อมูลและทำความรู้จักกับ Access

เวลาเรียนโดยประมาณ: 4 ชั่วโมง

เป้าหมายบทเรียน:

  • นักเรียนเข้าใจความหมายและความสำคัญของฐานข้อมูล
  • นักเรียนรู้จักโปรแกรม Microsoft Access และส่วนประกอบพื้นฐาน
  • นักเรียนสามารถระบุความแตกต่างระหว่างการเก็บข้อมูลแบบทั่วไปกับการเก็บข้อมูลแบบฐานข้อมูล

1.1 ฐานข้อมูลคืออะไร? ทำไมเราถึงต้องใช้?

ลองนึกภาพถึงสมุดพกของนักเรียน ที่มีข้อมูลชื่อ-นามสกุล, ชั้นเรียน, เลขที่, ผลการเรียนต่างๆ หรือสมุดโทรศัพท์ที่มีชื่อ, เบอร์โทรศัพท์, ที่อยู่

ฐานข้อมูล (Database) ก็คล้ายๆ กันครับ แต่เป็นที่เก็บข้อมูลจำนวนมหาศาลอย่างเป็นระบบระเบียบ สามารถจัดเก็บ, ค้นหา, แก้ไข และลบข้อมูลได้อย่างรวดเร็วและมีประสิทธิภาพ

ตัวอย่างง่ายๆ ของฐานข้อมูลในชีวิตประจำวัน:

  • ทะเบียนนักเรียนของโรงเรียนเรา: มีชื่อ, เลขประจำตัว, ที่อยู่, ผลการเรียน ฯลฯ
  • ระบบยืม-คืนหนังสือในห้องสมุด: มีข้อมูลหนังสือ, สมาชิก, วันที่ยืม-คืน
  • ข้อมูลสินค้าในร้านค้าสะดวกซื้อ: มีชื่อสินค้า, ราคา, จำนวน
  • รายชื่อผู้ติดต่อในโทรศัพท์มือถือของเรา: มีชื่อ, เบอร์โทร, อีเมล
  • ข้อมูลผู้ใช้ Facebook, Instagram: มีชื่อ, รูปโปรไฟล์, เพื่อน, โพสต์

ทำไมเราต้องใช้ฐานข้อมูล?

ลองคิดดูนะครับว่า ถ้าเรามีข้อมูลเยอะๆ แล้วเก็บแบบไม่เป็นระบบ เช่น เขียนใส่กระดาษบ้าง เก็บใน Excel บ้าง จะเกิดอะไรขึ้น?

  • หาสิ่งที่ต้องการยาก: ถ้าเราอยากรู้ว่านักเรียนชื่อ "สมใจ" อยู่ห้องไหน ต้องเปิดสมุดพกหลายเล่ม หรือเปิดไฟล์ Excel หลายไฟล์
  • ข้อมูลซ้ำซ้อน: อาจมีชื่อ "สมใจ" อยู่หลายที่ ทำให้ไม่แน่ใจว่าข้อมูลไหนถูกต้อง
  • ผิดพลาดง่าย: การแก้ไขข้อมูลที่กระจัดกระจายหลายที่ อาจลืมแก้บางที่ ทำให้ข้อมูลไม่ตรงกัน
  • จัดการยาก: การนำข้อมูลมารวมกันเพื่อวิเคราะห์ทำได้ยาก

ประโยชน์ของการใช้ฐานข้อมูล:

  • จัดเก็บข้อมูลอย่างเป็นระบบ: ทำให้ง่ายต่อการค้นหาและเรียกใช้
  • ลดความซ้ำซ้อนของข้อมูล: ข้อมูลแต่ละชุดจะถูกเก็บไว้เพียงที่เดียว
  • ป้องกันความผิดพลาดของข้อมูล: เมื่อแก้ไขข้อมูล ก็จะแก้ไขที่เดียว ทุกส่วนที่ใช้ข้อมูลนั้นก็จะอัปเดตตาม
  • รักษาความถูกต้องของข้อมูล: กำหนดเงื่อนไขการป้อนข้อมูลได้ เช่น ช่องเบอร์โทรศัพท์ต้องเป็นตัวเลขเท่านั้น
  • เพิ่มความรวดเร็วในการทำงาน: ค้นหา, จัดเรียง, สรุปข้อมูลได้ในเวลาอันสั้น
  • ความปลอดภัยของข้อมูล: สามารถกำหนดสิทธิ์การเข้าถึงข้อมูลได้

กิจกรรมชวนคิด:

"นักเรียนคิดว่าการเก็บข้อมูลลูกค้าในร้านกาแฟด้วยสมุดจด กับการใช้โปรแกรมคอมพิวเตอร์แบบฐานข้อมูล มีข้อดีข้อเสียต่างกันอย่างไร?"

กิจกรรมกลุ่ม: แบ่งกลุ่มนักเรียน ลองคิดตัวอย่างฐานข้อมูลง่ายๆ ในชีวิตประจำวัน หรือในโรงเรียนมากลุ่มละ 1 ตัวอย่าง แล้วนำเสนอ

1.2 รู้จักกับ Microsoft Access 2021

วันนี้เราจะมาทำความรู้จักกับโปรแกรมที่เราจะใช้เรียนรู้เรื่องฐานข้อมูล นั่นก็คือ Microsoft Access 2021 ครับ

Microsoft Access เป็นโปรแกรมจัดการฐานข้อมูล (Database Management System: DBMS) ที่พัฒนาโดย Microsoft อยู่ในชุด Microsoft Office เหมือนกับ Word, Excel, PowerPoint ครับ Access เหมาะสำหรับการสร้างฐานข้อมูลขนาดเล็กถึงขนาดกลาง ที่สามารถจัดการข้อมูลได้อย่างง่ายดาย

ส่วนประกอบสำคัญของฐานข้อมูล Access:

เปรียบเทียบกับบ้านของเรานะครับ:

  • ฐานข้อมูล (Database File): เหมือนกับ "บ้าน" ทั้งหลัง ที่เก็บทุกอย่างไว้ในที่เดียว ไฟล์ Access จะมีนามสกุล .accdb
  • ตาราง (Tables): เหมือนกับ "ห้อง" ต่างๆ ในบ้าน (ห้องนอน, ห้องครัว, ห้องน้ำ) เป็นส่วนที่ใช้เก็บข้อมูลจริงๆ ข้อมูลจะถูกจัดเก็บในรูปแบบแถว (Record) และคอลัมน์ (Field)
  • แบบสอบถาม (Queries): เหมือนกับ "สูตรอาหาร" หรือ "รายการของ" ที่เราไปหยิบของในครัว เป็นเครื่องมือที่เราใช้ค้นหา ดึงข้อมูลที่ต้องการจากตารางตั้งแต่หนึ่งตารางขึ้นไป
  • ฟอร์ม (Forms): เหมือนกับ "หน้าต่าง" หรือ "ประตู" ของบ้าน เป็นส่วนที่ใช้สำหรับป้อนข้อมูล แสดงข้อมูล หรือแก้ไขข้อมูล ให้ดูสวยงามและใช้งานง่ายขึ้น
  • รายงาน (Reports): เหมือนกับ "อัลบั้มรูป" หรือ "รายงานประจำปี" เป็นเครื่องมือที่ใช้สำหรับแสดงข้อมูลจากตารางหรือแบบสอบถามในรูปแบบที่สวยงาม เพื่อพิมพ์ออกมา หรือนำไปนำเสนอ

หน้าจอหลักของ Microsoft Access 2021

การเปิดโปรแกรม Microsoft Access 2021 และสำรวจหน้าจอ:

ขั้นตอนการเปิดโปรแกรม:

  1. คลิกปุ่ม Start (มุมล่างซ้ายของหน้าจอ)
  2. ค้นหา "Access" หรือเลื่อนหาในรายการโปรแกรม
  3. คลิกที่ "Access 2021"

เมื่อเปิดโปรแกรมขึ้นมา:

  • หน้าจอเริ่มต้น (Backstage View): จะเห็นตัวเลือกให้สร้างฐานข้อมูลเปล่า (Blank Database), เปิดไฟล์ที่มีอยู่แล้ว หรือเลือก Template สำเร็จรูป
  • แถบ Ribbon: เหมือนกับ Word, Excel, PowerPoint มีกลุ่มคำสั่งต่างๆ เช่น Home, Create, External Data, Database Tools
  • บานหน้าต่างนำทาง (Navigation Pane): อยู่ทางด้านซ้ายมือ จะแสดงรายชื่อของวัตถุทั้งหมดในฐานข้อมูลของเรา (Tables, Queries, Forms, Reports)
  • พื้นที่ทำงาน: พื้นที่ตรงกลางที่เราจะใช้สร้างและแก้ไขวัตถุต่างๆ

กิจกรรมสำรวจ:

ให้นักเรียนเปิดโปรแกรม Microsoft Access ขึ้นมาสำรวจหน้าจอต่างๆ และลองคลิกดูเมนูบน Ribbon

คำถามชวนคิด:

"จากที่ได้สำรวจ Access คิดว่าส่วนไหนสำคัญที่สุดในการเก็บข้อมูล และทำไม?"

บทที่ 2: การสร้างและจัดการตาราง (Tables)

เวลาเรียนโดยประมาณ: 4 ชั่วโมง

เป้าหมายบทเรียน:

  • นักเรียนเข้าใจแนวคิดของ Field, Record, Data Type และ Primary Key
  • นักเรียนสามารถออกแบบตารางที่เหมาะสมกับข้อมูลได้
  • นักเรียนสามารถสร้างตารางใน Access ได้
  • นักเรียนสามารถป้อนข้อมูล, แก้ไขข้อมูล และลบข้อมูลในตารางได้

2.1 การออกแบบตาราง: Field, Record, Data Type และ Primary Key

"ตาราง" คือหัวใจสำคัญของฐานข้อมูล Access เป็นที่ที่เราจะเก็บข้อมูลจริงๆ ครับ

ส่วนประกอบของตาราง:

  • ฟิลด์ (Field): เปรียบเสมือน "หัวข้อ" หรือ "คอลัมน์" ของข้อมูล เช่น ฟิลด์ "ชื่อ", ฟิลด์ "นามสกุล", ฟิลด์ "วันเกิด" แต่ละฟิลด์จะเก็บข้อมูลชนิดเดียวกัน
  • เรคอร์ด (Record): เปรียบเสมือน "แถว" หรือ "รายการ" ของข้อมูลทั้งหมดสำหรับ 1 หน่วย เช่น ข้อมูลของนักเรียน 1 คน (ชื่อ, นามสกุล, วันเกิด) ถือเป็น 1 เรคอร์ด

ตัวอย่างตาราง "นักเรียน":

รหัสนักเรียน ชื่อ นามสกุล วันเกิด เพศ ชั้น
65001 สมใจ ใจดี 1/1/2548 หญิง ม.5/1
65002 มานะ เก่งกาจ 5/3/2548 ชาย ม.5/1
65003 สุดา อ่อนหวาน 10/7/2548 หญิง ม.5/2

ในตัวอย่างนี้: "รหัสนักเรียน, ชื่อ, นามสกุล, วันเกิด, เพศ, ชั้น" คือ "ฟิลด์" ส่วน "แถวทั้งหมดของ สมใจ ใจดี" คือ "เรคอร์ด"

ชนิดข้อมูล (Data Types):

ฟิลด์แต่ละฟิลด์จะต้องกำหนด "ชนิดข้อมูล" เพื่อบอก Access ว่าข้อมูลในฟิลด์นั้นๆ เป็นข้อมูลประเภทไหน เพื่อให้ Access จัดการได้อย่างถูกต้องและลดข้อผิดพลาด

  • Short Text: ข้อความสั้นๆ หรือตัวเลขที่ไม่คำนวณ (เช่น ชื่อ, เบอร์โทรศัพท์) - ไม่เกิน 255 ตัวอักษร
  • Long Text: ข้อความยาวๆ (เช่น รายละเอียด)
  • Number: ตัวเลขที่นำไปคำนวณ (เช่น จำนวน, คะแนน)
  • Date/Time: วันที่และเวลา (เช่น วันเกิด)
  • Currency: ตัวเลขที่เป็นสกุลเงิน (เช่น ราคา)
  • AutoNumber: เลขลำดับที่ Access สร้างให้อัตโนมัติ ไม่ซ้ำกัน (เหมาะเป็น Primary Key)
  • Yes/No: ข้อมูลที่มีสองสถานะ (เช่น ใช่/ไม่ใช่)
  • Lookup Wizard: สร้างช่องให้เลือกข้อมูลจากรายการที่กำหนดเอง หรือจากตารางอื่น (สะดวกมาก!)
  • และอื่นๆ อีกหลายชนิด!

Primary Key (คีย์หลัก):

Primary Key คือ "ฟิลด์" หรือ "กลุ่มของฟิลด์" ที่มีค่าไม่ซ้ำกันในแต่ละเรคอร์ด ใช้สำหรับระบุแต่ละเรคอร์ดในตารางได้อย่างเฉพาะเจาะจง

  • สำคัญมาก! เพราะ Primary Key ทำให้ Access รู้ว่าเรคอร์ดไหนเป็นเรคอร์ดไหน และใช้ในการเชื่อมโยงข้อมูลระหว่างตาราง
  • ตัวอย่าง: รหัสนักเรียน, เลขประจำตัวประชาชน, รหัสสินค้า, เลขที่ใบเสร็จ
  • ข้อควรจำ: ต้องมีค่าไม่ซ้ำกัน และห้ามเป็นค่าว่าง (Null)

Workshop 1.1: ออกแบบตาราง "ข้อมูลนักเรียน" และ "ข้อมูลวิชา"

แบ่งกลุ่มนักเรียน (2-3 คน) ลองระดมความคิดว่าในตาราง "ข้อมูลนักเรียน" ควรมีฟิลด์อะไรบ้าง และกำหนดชนิดข้อมูลที่เหมาะสม พร้อมเลือกฟิลด์ที่จะเป็น Primary Key พร้อมเหตุผล

ทำเช่นเดียวกันสำหรับตาราง "ข้อมูลวิชา" (เช่น รหัสวิชา, ชื่อวิชา, หน่วยกิต, จำนวนชั่วโมง)

นำเสนอผลการออกแบบของแต่ละกลุ่ม

2.2 การสร้างตารางใน Access

เมื่อเราออกแบบตารางในกระดาษแล้ว ขั้นต่อไปคือการสร้างตารางจริงๆ ใน Access ครับ

วิธีสร้างตารางใน Access:

มี 2 มุมมองหลักๆ ในการสร้างตาราง:

  1. มุมมอง Datasheet View (มุมมองแผ่นข้อมูล):

    เหมาะสำหรับการสร้างตารางอย่างรวดเร็ว โดยป้อนชื่อฟิลด์และชนิดข้อมูลแบบคร่าวๆ จากนั้นจึงกลับไปแก้ไขรายละเอียดใน Design View

    • ไปที่แท็บ Create > คลิก Table
    • Access จะสร้างตารางเปล่าให้ พร้อมฟิลด์ ID (AutoNumber) และฟิลด์ "Click to Add"
    • ดับเบิลคลิกที่ "Click to Add" แล้วเลือกชนิดข้อมูล จากนั้นตั้งชื่อฟิลด์

  2. มุมมอง Design View (มุมมองออกแบบ):

    เหมาะสำหรับการออกแบบตารางอย่างละเอียด กำหนดชื่อฟิลด์, ชนิดข้อมูล, คำอธิบาย, กำหนด Primary Key และคุณสมบัติอื่นๆ ของฟิลด์

    • ไปที่แท็บ Create > คลิก Table Design
    • จะเห็น 3 คอลัมน์: Field Name (ชื่อฟิลด์), Data Type (ชนิดข้อมูล), Description (คำอธิบาย)
    • ป้อนชื่อฟิลด์และเลือกชนิดข้อมูล
    • คลิกขวาที่ชื่อฟิลด์ที่ต้องการให้เป็น Primary Key แล้วเลือก Primary Key (จะมีรูปกุญแจปรากฏขึ้น)
    • เมื่อสร้างฟิลด์ครบแล้ว ให้บันทึกตารางโดยคลิกปุ่ม Save (รูปแผ่นดิสก์) หรือกด Ctrl+S ตั้งชื่อตาราง (เช่น tblStudents, tblCourses)

ข้อแนะนำ:

ควรตั้งชื่อฟิลด์และตารางให้สื่อความหมาย โดยไม่มีช่องว่าง (Space) อาจใช้ขีดล่าง _ แทน เช่น StudentID, FirstName, CourseName

ควรใช้มุมมอง Design View ในการสร้างตารางเป็นหลัก เพื่อความละเอียดในการกำหนดคุณสมบัติ

2.3 การป้อนข้อมูลและจัดการข้อมูลในตาราง

เมื่อเราสร้างตารางเสร็จแล้ว ก็ถึงเวลาป้อนข้อมูลจริงลงไปครับ

การป้อนข้อมูลในตาราง:

  • เปิดตารางในมุมมอง Datasheet View (ดับเบิลคลิกที่ชื่อตารางใน Navigation Pane)
  • จะเห็นตารางเปล่าๆ มีหัวคอลัมน์เป็นชื่อฟิลด์ที่เราสร้างไว้
  • พิมพ์ข้อมูลลงในแต่ละช่อง (Cell) คล้ายกับการทำงานใน Excel
  • เมื่อป้อนข้อมูลในแถวแรกเสร็จแล้ว กด Tab เพื่อเลื่อนไปยังฟิลด์ถัดไป หรือ Enter เพื่อเลื่อนไปยังเรคอร์ดใหม่
  • ข้อมูลจะถูกบันทึกโดยอัตโนมัติเมื่อย้ายไปยังเรคอร์ดถัดไป

การแก้ไขข้อมูลในตาราง:

  • คลิกที่เซลล์ที่ต้องการแก้ไข
  • พิมพ์ข้อมูลใหม่ทับ หรือแก้ไขข้อมูลเดิม
  • กด Enter หรือคลิกที่เซลล์อื่นเพื่อยืนยันการแก้ไข

การลบข้อมูลในตาราง:

  • ลบเรคอร์ด (แถว):
    • คลิกที่ปุ่มเลือกเรคอร์ด (สี่เหลี่ยมเล็กๆ ด้านซ้ายสุดของแถว) เพื่อเลือกทั้งแถว
    • กดปุ่ม Delete บนคีย์บอร์ด หรือคลิกขวาที่แถวนั้น แล้วเลือก Delete Record
    • จะมีข้อความเตือนให้ยืนยันการลบข้อมูล (เมื่อลบแล้วจะกู้คืนไม่ได้)
  • ลบฟิลด์ (คอลัมน์): (ควรทำในมุมมอง Design View เพื่อไม่ให้ข้อมูลหาย)
    • ไปที่มุมมอง Design View
    • คลิกที่แถวฟิลด์ที่ต้องการลบ
    • กดปุ่ม Delete บนคีย์บอร์ด หรือคลิกขวาที่ฟิลด์แล้วเลือก Delete Rows
    • ข้อควรระวัง: การลบฟิลด์จะทำให้ข้อมูลทั้งหมดในฟิลด์นั้นหายไปอย่างถาวร!

การจัดเรียงและค้นหาข้อมูลเบื้องต้น:

  • การจัดเรียง (Sort): คลิกที่หัวคอลัมน์ (Field Header) แล้วเลือก Sort A to Z (น้อยไปมาก) หรือ Sort Z to A (มากไปน้อย)
  • การค้นหา (Find): กด Ctrl + F เพื่อเปิดกล่องค้นหา พิมพ์ข้อมูลที่ต้องการค้นหา

  • นักเรียนเข้าใจความจำเป็นของการสร้างความสัมพันธ์ระหว่างตาราง

3.1 การสร้างความสัมพันธ์ (Relationships) ระหว่างตาราง

เราได้เรียนรู้วิธีสร้างตารางแต่ละตารางแล้ว แต่ข้อมูลในโลกความเป็นจริงมักจะเกี่ยวข้องกัน เช่น นักเรียนเรียนวิชาต่างๆ, ครูสอนวิชาต่างๆ Access จะใช้ "ความสัมพันธ์" ในการเชื่อมโยงข้อมูลเหล่านี้เข้าด้วยกันครับ

ทำไมต้องสร้างความสัมพันธ์?

การสร้างความสัมพันธ์มีประโยชน์มาก เพราะ:

  • ลดความซ้ำซ้อนของข้อมูล: แทนที่จะต้องพิมพ์ชื่อวิชาซ้ำๆ ในตารางนักเรียน ก็แค่เก็บรหัสวิชา แล้วไปดึงชื่อวิชาจากตารางวิชาได้เลย
  • รักษาความถูกต้องของข้อมูล: เมื่อเปลี่ยนชื่อวิชาในตารางวิชา ข้อมูลในตารางอื่นๆ ที่เชื่อมโยงก็จะอัปเดตตามโดยอัตโนมัติ
  • สามารถดึงข้อมูลจากหลายตารางมาแสดงพร้อมกันได้: ซึ่งจำเป็นมากสำหรับการสร้าง Query, Form, และ Report ที่ซับซ้อนขึ้น

ชนิดของความสัมพันธ์ที่พบบ่อย: One-to-Many (หนึ่งต่อกลุ่ม)

  • One-to-Many: ข้อมูล 1 รายการในตารางแรก (ตารางหลัก) สามารถเชื่อมโยงกับข้อมูลได้หลายรายการในตารางที่สอง (ตารางรอง)
  • ตัวอย่าง: "ครู 1 คน" (ตารางครู) สามารถ "สอนได้หลายวิชา" (ตารางวิชา)
  • ตัวอย่าง: "วิชา 1 วิชา" (ตารางวิชา) สามารถมี "นักเรียนหลายคน" (ตารางนักเรียนที่ลงทะเบียน)

การสร้างความสัมพันธ์ใน Access:

  1. ปิดตารางทุกตารางที่เปิดอยู่ใน Access
  2. ไปที่แท็บ Database Tools > คลิก Relationships
  3. หากยังไม่มีตารางปรากฏ ให้คลิกขวาในพื้นที่ว่าง แล้วเลือก Show Table... เลือกตารางที่ต้องการนำมาสร้างความสัมพันธ์ (เช่น tblStudents, tblCourses) แล้วคลิก Add
  4. สร้างความสัมพันธ์แบบ One-to-Many:
    • ลากฟิลด์ Primary Key จากตารางหลัก (ฝั่ง "One", เช่น CourseID จาก tblCourses)
    • ไปวางบนฟิลด์ Foreign Key (ฟิลด์ที่ใช้เชื่อมโยงในตารางรอง, ฝั่ง "Many", เช่น CourseID ใน tblEnrollment)
    • หน้าต่าง Edit Relationships จะปรากฏขึ้น
    • เลือกช่อง Enforce Referential Integrity (สำคัญมาก! เพื่อบังคับใช้กฎความสัมพันธ์ เช่น ไม่สามารถลบวิชาที่มีนักเรียนลงทะเบียนอยู่ได้)
    • คลิก Create
  5. จะเห็นเส้นเชื่อมโยงระหว่างตาราง โดยมีเลข 1 ที่ฝั่ง One และสัญลักษณ์ Infinity (อนันต์) ที่ฝั่ง Many

ตัวอย่างหน้าต่าง Relationships ใน Access

ข้อควรจำ:

ในสร้างความสัมพันธ์แบบ One-to-Many ตาราง "Many" ต้องมีฟิลด์ที่ใช้เก็บค่า Primary Key ของตาราง "One" โดยฟิลด์นี้เรียกว่า Foreign Key

ก่อนสร้างความสัมพันธ์ ตารางต้องปิดทั้งหมด!

3.2 การสร้างแบบสอบถาม (Queries) เบื้องต้น

"แบบสอบถาม" หรือ "Query" เปรียบเหมือนคำถามที่เราถามฐานข้อมูล เพื่อให้ Access ดึงข้อมูลที่ต้องการมาให้เราดู หรือนำไปประมวลผลต่อครับ

แบบสอบถามคืออะไร? ทำไมต้องใช้?

  • Query คือเครื่องมือสำหรับดึงข้อมูลจากตารางเดียวหรือหลายตารางตามเงื่อนไขที่เรากำหนด
  • ประโยชน์:
    • ค้นหาข้อมูลเฉพาะ: เช่น ต้องการรายชื่อนักเรียนชายที่อยู่ ม.5/1 เท่านั้น
    • รวมข้อมูลจากหลายตาราง: เช่น ต้องการชื่อนักเรียนและชื่อวิชาที่นักเรียนคนนั้นลงทะเบียน
    • สรุปข้อมูล: เช่น นับจำนวนนักเรียนในแต่ละชั้น
    • คำนวณข้อมูล: เช่น คำนวณเกรดเฉลี่ย
    • ใช้เป็นแหล่งข้อมูลสำหรับ Form และ Report

ตัวอย่างหน้าจอ Query Design View

การสร้าง Select Query (แบบสอบถามเพื่อเลือกข้อมูล):

เราจะเน้นที่ Select Query ซึ่งเป็น Query ชนิดที่ใช้บ่อยที่สุด

  1. วิธีที่ 1: ใช้ Query Wizard (เหมาะสำหรับมือใหม่)

    • ไปที่แท็บ Create > คลิก Query Wizard
    • เลือก Simple Query Wizard > OK
    • เลือกตาราง/Query: เลือกตารางหรือ Query ที่ต้องการใช้
    • เลือกฟิลด์: เลือกฟิลด์ที่ต้องการให้แสดงใน Query
    • คลิก Next >
    • กำหนดชื่อ Query: ตั้งชื่อ Query (เช่น qryStudentList) > Finish

  2. วิธีที่ 2: ใช้ Query Design (เหมาะสำหรับการสร้าง Query ที่ซับซ้อนกว่า)

    • ไปที่แท็บ Create > คลิก Query Design
    • Show Table Dialog Box: เลือกตารางที่ต้องการใช้ > Add > Close
    • เพิ่มฟิลด์ลงใน Query: ดับเบิลคลิกที่ชื่อฟิลด์ หรือลากฟิลด์จากตารางลงมาใส่ในช่อง Field ด้านล่าง
    • ช่องต่างๆ ใน Query Design Grid:
      • Field: ชื่อฟิลด์ที่ต้องการแสดง
      • Table: ชื่อตารางที่ฟิลด์นั้นมาจาก
      • Sort: สำหรับจัดเรียงข้อมูล (Ascending: น้อยไปมาก, Descending: มากไปน้อย)
      • Show: ติ๊กเครื่องหมายถูก หากต้องการให้แสดงฟิลด์นั้นในผลลัพธ์
      • Criteria: ช่องสำหรับกำหนดเงื่อนไข (Condition) ในการเลือกข้อมูล

การกำหนดเงื่อนไข (Criteria) ใน Query:

เงื่อนไขจะช่วยให้เรากรองข้อมูลได้ เช่น อยากเห็นเฉพาะนักเรียน ม.5/1

  • ตัวอย่างเงื่อนไข:
    • เท่ากับ: "ม.5/1" (สำหรับ Text) หรือ 100 (สำหรับ Number)
    • มากกว่า/น้อยกว่า: >10000 (มากกว่า 10000), <50 (น้อยกว่า 50)
    • ระหว่าง: Between #1/1/2023# And #31/12/2023# (สำหรับ Date/Time)
    • มีคำว่า: Like "*สมใจ*" (มีคำว่า "สมใจ" อยู่ในข้อมูล)
    • ไม่ใช่: Not "หญิง"
    • เงื่อนไขหลายข้อ (AND): ใส่เงื่อนไขในบรรทัดเดียวกัน (เช่น `Class: "ม.5/1"` และ `Gender: "ชาย"`)
    • เงื่อนไขหลายข้อ (OR): ใส่เงื่อนไขคนละบรรทัดในช่อง "or:" (เช่น `Class: "ม.5/1"` (บรรทัดแรก) และ `Class: "ม.5/2"` (บรรทัด or))

ใช้ Query Wizard สร้าง Query ชื่อ qryAllStudents เพื่อแสดงฟิลด์ StudentID, FirstName, LastName, Class จาก tblStudents

Workshop 3.4: สร้าง Select Query พร้อมกำหนดเงื่อนไข

ใช้ Query Design สร้าง Query ชื่อ qryMaleStudentsM5_1

นำตาราง tblStudents เข้ามา

เพิ่มฟิลด์ StudentID, FirstName, LastName, Gender, Class

ในช่อง Criteria ของฟิลด์ Gender ใส่ "ชาย" และในช่อง Criteria ของฟิลด์ Class ใส่ "ม.5/1"

Workshop 3.5: สร้าง Select Query จากหลายตาราง

นำตาราง tblStudents, tblEnrollments, tblCourses เข้ามา

การบ้าน/ใบงาน:

  1. สร้าง Query ชื่อ qryFemaleStudents แสดงชื่อนักเรียนหญิงทั้งหมด
  2. สร้าง Query ชื่อ qryStudentsBorn2008 แสดงชื่อนักเรียนที่เกิดในปี 2008
  3. (ท้าทาย) สร้าง Query ชื่อ qryCourseEnrollmentCount ที่นับจำนวนนักเรียนที่ลงทะเบียนในแต่ละวิชา (ใช้ Group By และ Count)

ตอนนี้เราจะนำความรู้ทั้งหมดมาสร้างระบบฐานข้อมูลที่ใช้งานได้จริง ซึ่งจะช่วยให้โรงเรียนหรือครูที่ปรึกษาชมรมสามารถจัดการข้อมูลสมาชิกชมรม, ชมรมต่างๆ, กิจกรรม และการเข้าร่วมกิจกรรมของนักเรียนได้อย่างเป็นระบบ

1. การออกแบบตารางข้อมูล (Tables)

เราจะออกแบบ 5 ตารางหลักๆ เพื่อจัดเก็บข้อมูล

1.1. tblStudents (ข้อมูลนักเรียน)

  • วัตถุประสงค์: เก็บข้อมูลพื้นฐานของนักเรียนที่เป็นสมาชิกชมรม
  • ฟิลด์ (Fields):
    Field NameData TypeDescription

    EmailShort Textอีเมล

1.2. tblClubs (ข้อมูลชมรม)

  • วัตถุประสงค์: เก็บข้อมูลของชมรมต่างๆ ในโรงเรียน
  • ฟิลด์ (Fields):
    Field NameData TypeDescription

1.3. tblClubMembers (ข้อมูลสมาชิกชมรม)

  • วัตถุประสงค์: บันทึกว่านักเรียนคนใดเป็นสมาชิกชมรมใดบ้าง
  • ฟิลด์ (Fields):
    Field NameData TypeDescription

1.4. tblActivities (ข้อมูลกิจกรรมชมรม)

  • วัตถุประสงค์: เก็บข้อมูลกิจกรรมที่แต่ละชมรมจัดขึ้น

2. การสร้างความสัมพันธ์ (Relationships)

เราจะสร้างความสัมพันธ์แบบ One-to-Many ดังนี้:

  1. tblStudents (StudentID) -- 1:M -- tblClubMembers (StudentID)
  2. tblClubs (ClubID) -- 1:M -- tblClubMembers (ClubID)
  3. tblClubs (ClubID) -- 1:M -- tblActivities (ClubID)
  4. tblActivities (ActivityID) -- 1:M -- tblActivityAttendance (ActivityID)
  5. tblStudents (StudentID) -- 1:M -- tblActivityAttendance (StudentID)

แผนภาพความสัมพันธ์ของตารางในระบบจัดการข้อมูลชมรมโรงเรียน

3. การสร้างคิวรี (Queries)

เราจะสร้างคิวรีที่สำคัญ 4 ข้อ เพื่อดึงข้อมูลตามที่เราต้องการ

3.1. qryStudentClubMembership (ข้อมูลนักเรียนและชมรมที่สังกัด)

  • วัตถุประสงค์: แสดงข้อมูลนักเรียนและชมรมที่นักเรียนเป็นสมาชิก
  • ฟิลด์ที่ใช้: StudentID, FirstName, LastName, Class (จาก tblStudents), ClubName, AdvisorName (จาก tblClubs)

3.2. qryClubActivityList (รายการกิจกรรมของแต่ละชมรม)

  • วัตถุประสงค์: แสดงรายการกิจกรรมของแต่ละชมรม
  • ฟิลด์ที่ใช้: ClubName, ActivityName, ActivityDate, Location (จาก tblClubs, tblActivities)

3.3. qryStudentAttendanceSummary (สรุปการเข้าร่วมกิจกรรมของนักเรียน)

  • วัตถุประสงค์: แสดงว่านักเรียนแต่ละคนเข้าร่วมกิจกรรมอะไรบ้าง
  • ฟิลด์ที่ใช้: StudentID, FirstName, LastName (จาก tblStudents), ClubName (จาก tblClubs), ActivityName, ActivityDate (จาก tblActivities)

3.4. qryClubMembersCount (จำนวนสมาชิกในแต่ละชมรม)

  • วัตถุประสงค์: นับจำนวนสมาชิกในแต่ละชมรม
  • ฟิลด์ที่ใช้: ClubName (จาก tblClubs), และ CountOfStudents (ใช้ฟังก์ชัน Count ใน Query)

  • รูปแบบ: ฟอร์มแบบเดี่ยว (Single Form)
  • ปุ่มควบคุม: เพิ่มเรคอร์ดใหม่, บันทึก, ลบ, ค้นหา, ปิดฟอร์ม

สรุปขั้นตอนการสร้างระบบใน Microsoft Access (Step-by-Step)

นี่คือขั้นตอนที่เราใช้สร้างฐานข้อมูลนี้ใน Access จริง ๆ

  1. สร้างฐานข้อมูลใหม่:
    • เปิด Access เลือก **Blank desktop database** ตั้งชื่อไฟล์ `SchoolClubManagement.accdb`
  2. สร้างตาราง (Tables):
    • ใช้ **Create > Table Design** สร้างตาราง tblStudents, tblClubs, tblClubMembers, tblActivities, tblActivityAttendance ตามที่ออกแบบไว้
    • กำหนด Primary Key และชนิดข้อมูลให้ถูกต้องทุกฟิลด์
    • บันทึกแต่ละตาราง
  3. สร้างความสัมพันธ์ (Relationships):
    • ปิดตารางทั้งหมดก่อน!
    • ไปที่ **Database Tools > Relationships**
    • ลากฟิลด์ Primary Key ไปยัง Foreign Key ที่เชื่อมโยงกัน และเลือก **Enforce Referential Integrity**
    • บันทึก Layout
  4. ป้อนข้อมูลตัวอย่าง:
    • เปิด tblStudents และ tblClubs ใน Datasheet View แล้วป้อนข้อมูลสมมติอย่างน้อย 3-5 รายการ
  5. สร้างคิวรี (Queries):
    • ใช้ **Create > Query Design** สร้างคิวรี qryStudentClubMembership, qryClubActivityList, qryStudentAttendanceSummary, qryClubMembersCount
    • ตรวจสอบผลลัพธ์ของแต่ละคิวรี
  6. สร้างฟอร์ม (Forms):
    • ใช้ **Create > Form Wizard** หรือ **Form Design** เพื่อสร้างฟอร์ม frmStudents, frmClubs, frmClubMembers, frmActivities, frmActivityAttendance
    • สำคัญ: ในฟอร์ม frmClubMembers, frmActivities, frmActivityAttendance ให้ใช้ **Combo Box (Lookup Wizard)** เพื่อเลือกข้อมูลจากตารางอื่น
    • เพิ่มปุ่มควบคุม (Add New, Save, Delete, Close) บนแต่ละฟอร์ม
  7. สร้างรายงาน (Reports):
    • ใช้ **Create > Report Wizard** สร้างรายงาน rptStudentList, rptClubMembersByClub, rptActivitiesSummary
    • ลองดูตัวอย่างรายงาน (Print Preview)
  8. สร้างหน้าหลัก (Main Menu Form):
    • ใช้ **Create > Blank Form** ตั้งชื่อว่า frmMainMenu
    • ใน Design View, เพิ่ม Label สำหรับชื่อระบบ และ Command Buttons ต่างๆ
    • ใช้ **Command Button Wizard** หรือเขียน VBA Code ใน Event 'On Click' ของปุ่ม เพื่อให้ปุ่มเปิดฟอร์มหรือรายงานที่เกี่ยวข้อง

      • ตัวอย่างโค้ด VBA สำหรับปุ่มเปิดฟอร์ม/รายงาน:

      
Private Sub cmdOpenStudents_Click()
    DoCmd.OpenForm "frmStudents", acNormal
End Sub

Private Sub cmdViewStudentReport_Click()
    DoCmd.OpenReport "rptStudentList", acViewPreview
End Sub

Private Sub cmdExitSystem_Click()
    Application.Quit
End Sub
    • ปรับแต่งหน้าตาของ frmMainMenu ให้สวยงาม
  9. ตั้งค่าเริ่มต้นของฐานข้อมูล:
    • ไปที่ **File > Options > Current Database**
    • ในส่วน **Display Form** ให้เลือก `frmMainMenu`
    • คลิก **OK** แล้วปิด Access และเปิดใหม่ เพื่อดูว่าหน้าหลักแสดงขึ้นมาเองหรือไม่
  10. ทดสอบระบบ:
    • ลองคลิกปุ่มต่างๆ บนหน้าหลัก
    • ป้อนข้อมูลผ่านฟอร์ม, ลองแก้ไข, ลบ
    • ตรวจสอบผลลัพธ์ของคิวรีและรายงาน

คำแนะนำ:

  • ลงมือทำเยอะๆ: การเรียนรู้ที่ดีที่สุดคือการลงมือปฏิบัติจริง ลองผิดลองถูกด้วยตัวเอง
  • อย่ากลัวผิด: การทำผิดคือส่วนหนึ่งของการเรียนรู้ แก้ไขข้อผิดพลาดไปด้วยกัน
  • ถามคำถาม: ถ้าสงสัยหรือไม่เข้าใจตรงไหน ให้ถามคุณครูหรือเพื่อน ๆ ทันที
  • เน้นความเข้าใจ: ไม่ต้องรีบร้อน แต่ละขั้นตอนมีความสำคัญต่อภาพรวมของระบบ
  • ลองสร้างระบบของตัวเอง: เมื่อทำระบบชมรมได้แล้ว ลองคิดระบบอื่น ๆ ที่สนใจ เช่น ระบบยืม-คืนอุปกรณ์กีฬา, ระบบเก็บข้อมูลเกม ฯลฯ
  • ประสบการณ์: ฝึกออกแบบเยอะ ได้คิดเยอะ ลงมือทำเยอะๆ ก็จะมีประสบการณ์ แต่ไม่ได้หมายความต้องทำทั้งหมดในสิ่งที่คิด
